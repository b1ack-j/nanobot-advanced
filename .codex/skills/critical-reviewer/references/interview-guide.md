# Interview Guide: Review Calibration

Light interview before reviewing. 1-3 questions to focus the review.

## Risk Appetite

- How conservative should this review be?
  - Conservative: flag everything that could go wrong
  - Balanced: focus on likely risks with significant impact
  - Aggressive: only flag deal-breakers, accept higher risk for speed

## Focus Areas

- What concerns you most about this plan?
  - Technical feasibility
  - Cost / ROI
  - Timeline realism
  - Specific domain risks
  - Operational complexity
  - Something else?

## Review Outcome

- If the review finds significant problems, what would you prefer?
  - Loop back and revise the affected phase docs
  - Proceed with noted conditions and mitigations
  - Pause and discuss before deciding

## Perspective Selection

- Which expert perspectives matter most for this project?
  - Domain expert (e.g., trading, healthcare, fintech)
  - System architect
  - Risk analyst
  - Economics / feasibility
  - Operations / reliability
  - User / stakeholder advocate
  - All of the above (default)

## Best Practices

1. Keep calibration to 1-2 turns max -- don't over-interview before reviewing
2. Default to "balanced" review if user doesn't have strong preference
3. Use focus areas to prioritize which perspectives to go deeper on
4. If user says "just review it", use balanced defaults and cover all perspectives
